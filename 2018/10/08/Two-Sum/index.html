Two Sum

2018-10-08
LeetCodeEasy
Hash

First Leetcode exercise, keep going!

Description

Given an array of integers, return indices of the two numbers such that they add up to a specific target.

You may assume that each input would have exactly one solution, and you may not use the same element twice.

Example:

Solution

Since the array is not in order, we can not use the idea of two pointers at the begining and end. To make it in $O(n)$, we just go through the array once and store the information we need.

List

The basic idea is to check whether the current nums[i] and the recorded List[j] could added to the target. So we’d like to check target-nums[i] is in the list or not, if not, we append nums[i] to the list, if yes, we just return the index (j,i).

Dict

The only drawback of using List is the index() function is also $O(n)$, which means in the worse case, the runtime is $2n$. Instead, we can use a dict to store the number as well as the index. And the query is $O(1)$.

Analysis

The List method takes about 368ms to run the testcase while Dict mehod only use 36ms. We do see the power of Hash here.
